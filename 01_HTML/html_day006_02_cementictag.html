body 태그 내의 내용을 일반적으로 header / main / footer 의 세 부분으로 나누어 둠

cementic 태그는 선택이 아니라 필수이다.



cemantic 태그 cemantic

div 태그 대신에 의미를 전달할 수 있는 태그

cementic 태그들은 기본적으로 block이다.

텍스트, inline, block 을 모두 포함할 수 있다.

  • header : html 문서의 헤더 영역을 의미하는 태그(제목, 로고, 내비게이션, 검색창 등이 포함됨)
    텍스트, inline, block 모두 포함 가능하다. div와 거의 비슷한 느낌이다. header 태그나 footer 태그를 다시 들여오는 것은 불가능. 작동은 할 수 있으나 구조적인 문제가 있을 수 있다.
    헤더 사용법

  • main : html 문서의 주요 내용이 오는 태그
    텍스트, inline, block 모두 포함 가능함.
    메인 사용법

  • footer : html 문서의 푸터 영역을 의미하는 태그(저작권 정보, 관련 문서 링크, 패밀리 사이트, 주소, 영업시간, 사업자 번호, 전화번호 등이 포함됨)
    텍스트, inline, block 모두 포함 가능함. header 태그나 footer 태그를 다시 들여오는 것은 불가능. 작동은 할 수 있으나 구조적인 문제가 있을 수 있다.
    푸터 사용법


  • aside : 메인 컨텐츠와 직접적으로 관련이 없는 영역을 의미하는 태그. 홈페이지에서 마우스로 스크롤해서 화면이 움직일 때 같이 따라다니는 작은 창 같은 것을 aside 태그에 많이 집어넣는다. (SNS, 최근 본 상품, 광고 배너, 화면 상단으로 이동 등)
    보통은 html 문서의 오른쪽이나 왼쪽에 사이드 메뉴나 광고 등의 영역으로 사용한다.
    aside 사용법

  • nav : html 문서의 메인 메뉴나 목차 등을 정의해주는 태그. (보통 header 태그 내에서 메인 메뉴에 사용된다.)
    nav 사용법

  • section : html 문서 내에서 맥락이 같은 요소들을 주제별로 그룹화 해주는 태그.
    section 사용법

  • article : html 문서 내에서 독립적으로 배포 또는 재사용이 가능한 게시물, 뉴스기사, 블로그 포스팅을 나타낼 때 사용한다.
    텍스트, inline, block 모두 포함 가능함.
    article 사용법

  • address : footer 내에서 주소, 사업자 등록번호, 사업장 이름 등을 나타낼 때 사용한다.(footer 태그가 아니더라도 사용할 수 있으나 일반적으로 footer 안에서 많이 사용됨.)
    기본 스타일(p 태그 아래 위로 여백 생기는 것과 같은 느낌을 기본 스타일 이라고 한다.)로 italic체가 적용되어 있다.
    address 사용법
